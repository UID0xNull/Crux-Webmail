📖 Glosario

Términos técnicos usados en la documentación de Crux-Webmail.

A-C

Amavis
Content filter que conecta Postfix con ClamAV (antivirus) y SpamAssassin (anti-spam). Escanea cada email entrante.
ACME
Automated Certificate Management Environment. Protocolo usado por Let's Encrypt para provisioning automático de certificados TLS.
BullMQ
Job queue para Redis, usado en Fastify para tareas asíncronas (envío de email, processing de attachments).
ClamAV
Motor de antivirus open source usado por Amavis para escanear attachments.
CORS
Cross-Origin Resource Sharing. Mecanismo de seguridad para controlar qué dominios pueden hacer requests a la API.
DKIM
DomainKeys Identified Mail. Firma criptográfica en headers de email para verificar autenticidad.
DMARC
Domain-based Message Authentication, Reporting & Conformance. Política SPF+DKIM para reject/ quarantine de emails falsos.
DNSSEC
Domain Name System Security Extensions. Cifrado criptográfico de respuestas DNS contra spoofing.

D-M

Docker Compose
Herramienta de orchestration multi-container definida en YAML.
Dovecot
IMAP/POP3 server que maneja delivery de email a buzones y acceso de clientes.
Fastify
Framework web Node.js ultrarrápido usado como backend API de Crux-Webmail.
Grafana
Plataforma de visualización de métricas conectada a Prometheus como data source.
Hardening
Proceso de reducir la superficie de ataque de un sistema (read-only FS, cap_drop, etc.).
JMAP
JSON Meta Application Protocol. Standard RFC-8620, sucesor de IMAP/CardDAV/CalDAV con API REST JSON.
JWT
JSON Web Token. Token para autenticación stateless en la API.
LMTP
Local Mail Transfer Protocol. Protocolo para delivery de email desde Postfix → Dovecot.
mTLS
Mutual TLS. Ambos lados (client y server) se autentican con certificados.
Metric
Medición numérica recolectada por exporters de Prometheus (CPU, memoria, request count...).

N-S

Next.js
Framework React para frontend, con SSR, ISR, App Router.
Nginx
Reverse proxy y web server, front-door de Crux-Webmail. Maneja TLS termination y routing.
Postfix
MTA (Mail Transfer Agent) para envío y recepción de email vía SMTP.
PITR
Point-in-Time Recovery. Capacidad de restaurar PostgreSQL a un momento específico.
Prometheus
Sistema de monitoring time-series que scrapea métricas de exporters.
PWA
Progressive Web App. Instalable, offline-capable, con service worker y manifest.
RPO
Recovery Point Objective. Máximo tiempo de datos aceptable a perder.
RTO
Recovery Time Objective. Tiempo máximo de downtime aceptable.
Sequelize
ORM para PostgreSQL usado en el backend.
SPF
Sender Policy Framework. Registro DNS que autoriza IPs para enviar email de un dominio.

T-Z

Time-series DB
Base de datos optimizada para métricas temporales (Prometheus tsdb).
WebCrypto
API nativa del browser para operaciones criptográficas (E2E encryption).
WASM
WebAssembly. Runtime de baja confianza para sanitización de content.
WASM Sandbox
Ejecución de código WASM en un entorno aislado para sanitizar HTML y attachments.
WebSockets
Protocolo full-duplex para notificaciones en tiempo real (nuevos emails).
WAL
Write-Ahead Log. Logs de PostgreSQL para crash recovery y PITR.
Zero-Trust
Modelo de seguridad donde nada es confiado por defecto. Todo se autentica y autoriza.
