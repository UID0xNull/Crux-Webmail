Procedimiento de Actualización Paso a Paso

  1. Backup pre-update: docker exec crux-postgres pg_dump -U crux_user -d crux_mail > pre_update.sql
  2. Verificar cambios: git log --oneline HEAD...origin/main
  3. Pull + Build: git pull && npm run docker:build
  4. Deploy: npm run docker:up (equivalente a docker compose up -d)
  5. Monitorizar health: docker compose ps — esperar todos healthy
  6. Smoke test: curl -k https://localhost/api/health
  7. Verificar logs: docker compose logs --tail=100

Migraciones de Base de Datos

Las migraciones usan Sequelize y se ejecutan EN ANTES del deploy:

# Verificar migraciones pendientes
docker compose exec postgres psql -U crux_user -d crux_mail -c "SELECT * FROM SequelizeMeta ORDER ON migration_name DESC LIMIT 5;"

# Ejecutar migraciones
docker compose run --rm server npx sequelize db:migrate

# Verificar resultado
docker compose run --rm server npx sequelize db:migrate:status

⚠️ IMPORTANTE: Nunca hacer rollback de migraciones en producción. Solo aplicar forwards.

Automatización con Git Hooks

# Pre-deploy hook (ejemplo)
cat > .hooks/pre-deploy.sh << 'EOF'
#!/bin/bash
set -euo pipefail

# Backup automático pre-deploy
docker exec crux-postgres pg_basebackup -U crux_user -D /backups/pre-deploy -Ft -z -P

# Verificar que no haya secrets en diff
git diff --cached --name-only | grep -q '.secret\|\.key\|password' && { echo "Secrets detected! Abort."; exit 1; }

# Run lint + typecheck
npm run lint
npm run typecheck
npm run test:unit

echo "Pre-deploy checks PASSED"
EOF
