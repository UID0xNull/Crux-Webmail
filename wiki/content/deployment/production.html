🚀 Despliegue en Producción

Guía de despliegue basada en docker-compose.yml con 16 servicios, 4 redes segmentadas y secretos gestionados por Docker.

Requisitos del Servidor

  • OS: Linux (Debian 12 / Ubuntu 22.04+) — kernel 5.15+ para eBPF
  • RAM: 8 GB mínimo (ClamAV solo consume ~2 GB), 16 GB recomendado
  • Docker: 24+ con Compose v2+
  • Dominio: DNS configurable (records A/AAAA, MX, TXT)
  • Puertos: 25 (SMTP), 443 (HTTPS), 587 (submission), 3000 (Grafana dashboard)

Proceso de Despliegue Paso a Paso

  1. Clonar el repositorio
    git clone https://github.com/username/Crux-Webmail.git && cd Crux-Webmail
  2. Crear secretos en ./secrets/
    echo 'password_segura' > ./secrets/postgres_password.txt
    echo 'minio_password' > ./secrets/minio_password.txt
    echo 'grafana_admin_pass' > ./secrets/grafana_password.txt
    chmod 600 ./secrets/*.txt
  3. Generar certificados TLS
    mkdir -p ./certs && mkdir -p ./nginx/certs
    Usar Let's Encrypt (certbot) o certificados propios, copiar a ./nginx/certs/
  4. Configurar .env
    Copiar .env.example a .env y completar variables.
  5. Levantar stack completo
    docker compose -f docker-compose.yml up -d
  6. Verificar healthchecks
    docker compose ps — todos en (healthy)
  7. Crear usuario admin y seedear DB
    Ejecutar seeds/migraciones según configuración de Sequelize.
  8. Verificar conectividad
    curl -k https://localhost/api/health

Seguridad del Despliegue

Cada contenedor aplica hardening explícito:

  • read_only: true — filesystem inmutable (tmpfs para escritura temporal)
  • no-new-privileges: true — prevención de escalación
  • cap_drop: [ALL] — capabilities mínimo (solo las necesarias por servicio)
  • Secrets gestionados vía Docker secrets (archivos en ./secrets/)
  • Limits de memoria explícitos: ClamAV (2 GB), PostgreSQL (1 GB), Redis (512 MB), Prometheus (2 GB)
  • Log rotation automática: max 50 MB × 3 archivos por contenedor

Post-Deploy Checklist

✅ docker compose ps — 16 servicios (healthy)
✅ docker compose logs --tail=50 — sin errores críticos
✅ DNS configurado: A/AAAA, MX, SPF, DKIM, DMARC
✅ Prueba SMTP: telnet localhost 25 — EHLO responde
✅ Prueba HTTPS: curl -k https://localhost/health
✅ Grafana accesible: http://localhost:3000
✅ Prometheus targets: http://localhost:9090/targets — todos UP
✅ Backup configurado y schedule programado
