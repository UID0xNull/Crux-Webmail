Backup PostgreSQL (PITR)

# Backup base diario (cron: 02:00)
docker exec crux-postgres pg_basebackup \\
  -U crux_user \\
  -D /backups/base \\
  -Ft -z -P \\
  --checkpoint=fast

# Schema dump
docker exec crux-postgres pg_dump \\
  -U crux_user \\
  -d crux_mail \\
  --schema-only \\
  > /backups/schema_$(date +%F).sql

Backup Dovecot + Postfix

# Mail stores incremental
docker run --rm \\
  -v crux-webmail_dovecot_mail:/source:ro \\
  -v /backup/dovecot:/dest \\
  alpine tar czf /dest/dovecot_$(date +%F).tar.gz \\
  -C /source . --newer-mtime='1 day'

# Postfix mail
docker run --rm \\
  -v crux-webmail_postfix_mail:/source:ro \\
  -v /backup/postfix:/dest \\
  alpine tar czf /dest/postfix_$(date +%F).tar.gz \\
  -C /source .

Procedimiento de Restauración

  1. Parar servicios: docker compose stop postfix dovecot
  2. Restaurar PostgreSQL: docker exec -i crux-postgres pg_restore -U crux_user -d crux_mail
  3. Restaurar Dovecot mail: docker run --rm -v crux-webmail_dovecot_mail:/data -v /backup/dovecot:/b alpine tar xzf /b/dovecot_LATEST.tar.gz -C /data
  4. Restaurar Minio attachments: mc cp /backup/minio_LATEST.tar.gz myminio/attachments/
  5. Reiniciar: docker compose up -d
  6. Verificar: docker compose ps — todos healthy
