🔐 WebCrypto E2E — Cifrado del Lado del Cliente

Cifrado AES-GCM + RSA-OAEP nativo del browser.

Arquitectura de Cifrado

┌─────────────────────────────────────────────────────────┐
│              Client Side (Browser)                        │
│                                                         │
│  ┌───────────────────────────────────────────────────┐  │
│  │           WebCrypto API (window.crypto.subtle)    │  │
│  │                                                  │  │
│  │  1. Generate RSA-OAEP key pair (4096 bits)      │  │
│  │  2. Encrypt private key with password (PBKDF2)  │  │
│  │  3. Derive AES-GCM key from password             │  │
│  │  4. Encrypt email body: AES-GCM + random IV     │  │
│  │  5. Store encrypted data en IndexedDB            │  │
│  │  6. Send to server only encrypted payload        │  │
│  └───────────────────────────────────────────────────┘  │
│                                                         │
│  Server NEVER sees plaintext                            │
└─────────────────────────────────────────────────────────┘

Algoritmos

PropósitoAlgoritmoParámetros
Key derivationPBKDF2SHA-256, 100k iterations
Data encryptionAES-GCM256-bit keys, random 12-byte IV
Key exchangeRSA-OAEP4096-bit, SHA-256 padding
Key agreementECDHP-256 curve
SignaturesECDSAP-256 curve, SHA-256

Almacenamiento de Claves

// Key management con Web Crypto API
async function encryptData(password: string, data: string): Promise {
  // 1. Derivar key desde password
  const keyMaterial = await crypto.subtle.importKey(
    'raw', new TextEncoder().encode(password),
    'PBKDF2', false, ['deriveKey']
  );

  // 2. PBKDF2 derive AES key
  const aesKey = await crypto.subtle.deriveKey(
    { name: 'PBKDF2', salt: randomSalt, iterations: 100_000, hash: 'SHA-256' },
    keyMaterial,
    { name: 'AES-GCM', length: 256 },
    false,
    ['encrypt']
  );

  // 3. Encrypt
  const iv = crypto.getRandomValues(new Uint8Array(12));
  const ciphertext = await crypto.subtle.encrypt(
    { name: 'AES-GCM', iv }, aesKey, new TextEncoder().encode(data)
  );

  // 4. Store with salt + iv for decryption
  return JSON.stringify({ salt, iv, ciphertext });
}

Consideraciones de Seguridad

  • Keys nunca salen del browser: Private key RSA nunca se envía al server
  • Password never stored: Solo derivado PBKDF2, password memorizado por user
  • IV único por mensaje: Nunca reutilizar IV en AES-GCM
  • Secure context: WebCrypto requiere HTTPS (o localhost)
  • Key rotation: Manual rotation cada 90 días
  • Recovery: Exportar backup key encryptado con passphrase separado
