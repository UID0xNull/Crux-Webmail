🌐 JMAP Bridge — Capa de Protocolo Moderno

Traducción de peticiones JMAP RFC 8620 a operaciones IMAP/SMTP bajo el capó.

¿Qué es JMAP?

JMAP (JSON Meta Application Protocol) — RFC 8620 — es un protocolo moderno que reemplaza IMAP, SMTP y CalDAV. Usa HTTP POST con JSON, elimina las limitaciones del protocolo textual heredado.

Traducción de Peticiones

JMAPTraduce aDescripción
Email/getIMAP UID FETCHObtener emails con filtros
Email/setIMAP STORE + APPENDCambiar flags, crear mensajes
Mailbox/getIMAP LIST/LSUBObtener carpetas
Contact/getSequelize → PostgreSQLLeer contactos
CalendarEvent/getSequelize → PostgreSQLEventos de calendario
Account/getSequelize → PostgreSQLCuentas de usuario

Endpoint JMAP

POST /jmap
Content-Type: application/json
Authorization: Bearer {jwt_token}

{
  "using": ["urn:ietf:params:jmap:core","urn:ietf:params:jmap:mail"],
  "methodCalls": [
    [
      "Email/get",
      {
        "id": "1",
        "accountId": "usr_42",
        "fields": ["from","subject","dateSent"],
        "limit": 50
      },
      null
    ]
  ]
}

Ventajas sobre IMAP nativo

  • Paginación nativa: Sin necesidad de UIDs manuales
  • Batch operations: Múltiples métodos en una sola llamada
  • Change detection: PosChangeKey para sincronización eficiente
  • Filters nativos: Filtros en el servidor, no post-filtering local
  • HTTP semantics: Headers, caching, content negotiation
