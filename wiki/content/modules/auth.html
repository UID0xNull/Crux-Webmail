🔐 Sistema de Autenticación

JWT dual, refresh rotation y session invalidation.

Flujo de Autenticación

  ┌──────────┐     POST /api/auth/login     ┌──────────┐
  │  Client  │ ──────────────────────────────▶ │  Fastify │
  │ (Web/PWA)│                                │  Auth    │
  │          │ ◀────────────────────────────── │  Module  │
  └──────────┘   JWT + Refresh Token          └──────────┘
                                      │
                                      ▼
                                ┌──────────┐
                                │PostgreSQL│ ← bcrypt verify
                                │ (Users)  │    hash + salt
                                └──────────┘
                                      │
                                ┌──────────┐
                                │  Redis   │ ← Refresh token store
                                │ (Sesiones)│
                                └──────────┘

Configuración JWT

// fastify.config.js
module.exports = {
  jwt: {
    secret: process.env.JWT_SECRET,        // ≥ 256 bits
    cookie: false,                          // Sin cookies
    sign: {
      expiresIn: '5min',                   // Access token TTL
      algorithm: 'HS256',
    },
    verify: { maxAge: '5min' },
  },
  refreshTokens: {
    expiresIn: '7d',
    redisKeyPrefix: 'sess:',
    rotation: true,                         // New refresh on use
    invalidation: 'immediate',              // On logout
  },
};

Protecciones

  • Brute force: Rate limit 5/min en /login (Redis + Fastify rate-limit)
  • Account lockout: 5 fails → 15min lockout (exponential backoff)
  • Token binding: Refresh token bound a client_id (device fingerprint)
  • Revoke cascade: Logout → delete refresh en Redis + hash en DB
  • CORS restrict: Solo permitidos: origin del webmail
