Redis

Key Patterns

sess:{user_id}:{client_id}   → Refresh token (TTL: 7d)
cache:emails:{user_id}:{page} → Paginated emails (TTL: 60s)
rate:{ip}:{endpoint}         → Rate limit counter (TTL: 1min)
lock:{resource_id}           → Distributed locks (TTL: 30s)
q:send                       → Send queue (sorted set)
notif:{user_id}              → User notifications (TTL: 24h)
