💻 Cliente de Email — Next.js Frontend

Interfaz webmail SPA con SSR, PWA support y UI responsiva.

Estructura del Frontend

src/frontend/
├── app/
│   ├── (auth)/          # Login, register, forgot-password
│   ├── (main)/          # Dashboard, inbox, composer
│   │   ├── inbox/
│   │   ├── compose/
│   │   ├── calendar/
│   │   └── contacts/
│   ├── api/             # Route handlers (revalidación, etc.)
│   ├── layout.tsx       # Root layout, providers
│   └── page.tsx         # Landing / redirect
├── components/
│   ├── EmailList.tsx    # Virtualized list
│   ├── EmailView.tsx    # Render HTML/text con sandbox
│   ├── Composer.tsx     # Compose con attachments
│   ├── Sidebar.tsx      # Folders + quick search
│   └── shared/          # Buttons, modals, tooltips
├── lib/
│   ├── api.ts           # Fetch wrapper con JWT
│   ├── crypto.ts        # WebCrypto helpers
│   └── store.ts         # Zustand stores
├── styles/
│   └── globals.css      # Tailwind base
└── public/
    ├── manifest.json    # PWA manifest
    └── icons/           # Web app icons

Características

  • SSR/SSG: Login y landing pre-renderizados
  • PWA: Service worker con cache strategy (stale-while-revalidate)
  • Responsive: Mobile-first breakpoints (320px → 1920px)
  • Virtualización: React-window para listas de emails
  • Dark mode: CSS variables + class toggle en :root
  • Email render: Sandboxed iframe con CSP strict para HTML emails
  • Composer: Rich text editor con attachments drag & drop
