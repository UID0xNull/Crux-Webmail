⚙️ Backend de Email — Fastify Server

API REST + JMAP con validación Zod, rate limiting y observabilidad.

Estructura del Backend

src/server/
├── config/
│   ├── index.ts         # App config (Zod validation)
│   └── schema.ts        # Zod schemas para env vars
├── middleware/
│   ├── auth.ts          # JWT verification
│   ├── rateLimit.ts     # Redis-based rate limiting
│   └── errorHandler.ts  # Central error handling
├── routes/
│   ├── auth.routes.ts   # /api/auth/*
│   ├── email.routes.ts  # /api/emails/*
│   ├── user.routes.ts   # /api/users/*
│   └── health.routes.ts # /api/health/*
├── services/
│   ├── emailService.ts  # Email CRUD
│   ├── authService.ts   # Login, tokens, refresh
│   └── contactService.ts# Contact management
├── modules/
│   ├── imap-bridge.ts   # IMAP client pool
│   ├── smtp-bridge.ts   # SMTP send adapter
│   └── jmap-bridge.ts   # JMAP protocol translator
├── models/
│   ├── user.model.ts    # Sequelize User
│   ├── email.model.ts   # Sequelize Email
│   └── mailbox.model.ts # Sequelize Mailbox
└── app.ts              # Fastify factory

Middleware Pipeline

  1. Helmet: CSP, X-Frame-Options, X-Content-Type-Options
  2. CORS: Restrict origins al frontend + dev origins
  3. Pino Logger: Structured JSON logs con correlation ID
  4. Rate Limiting: Redis counter (100 req/min per IP, 5/min en auth)
  5. JWT Auth: fastify-jwt con decode + verify en cada endpoint
  6. Zod Validation: Schema validation en request body/query/params
  7. Error Handler: Central handler que retorna errores consistentes
