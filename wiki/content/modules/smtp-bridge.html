📤 SMTP Bridge — Módulo de Envío

Adaptador SMTP seguro para envío de emails a través de Postfix.

Arquitectura

El SMTP Bridge permite enviar emails desde la API Fastify hacia Postfix. Cada mensaje pasa por la cadena de filtros: DKIM signing → Amavis → Postfix queue.

  [Client] → Fastify API → [SMTP Bridge] → Postfix:25
       │                       │                │
       │  /api/send            │  MAIL FROM     │
       │                       │  RCPT TO       │
       │                       │  DATA          │
       │                       │                │
       │                       │  ┌──────────────┤
       │                       │  │ DKIM Sign    │
       │                       │  │ Amavis Scan  │
       │                       │  │ Greylist     │
       └─── JWT Auth ──────────┴──┴──────────────┘

Endpoint

MétodoEndpointDescripción
POST/api/sendEnviar email (body: from, to, cc, bcc, subject, text, html, attachments)
POST/api/send/batchEnvío masivo (max 50 por batch)
GET/api/sent/:queue-idEstado de un email en cola

Configuración

POSTFIX_HOST=172.21.0.10
POSTFIX_PORT=25
POSTFIX_HOSTNAME=mail.crux.local
POSTFIX_MESSAGE_SIZE_LIMIT=26214400

Validaciones

  • Tamaño: Límite de 25MB por mensaje (incluye attachments)
  • Recipients: Máximo 100 destinatarios por email
  • DKIM: Firma automática con selector "default" y Ed25519
  • Rate limit: 300 RPM en endpoints de autenticación
  • Spam check: Mensajes pasan por Amavis antes de salir

Cadena de Filtros de Salida

  1. SMTP Bridge: Recibe request de Fastify, valida estructura MIME
  2. OpenDKIM: Firma con milter (algoritmo ed25519)
  3. Amavis: ClamAV antivirus + SpamAssassin
  4. Postfix queue: Mensaje en cola hasta entrega final
