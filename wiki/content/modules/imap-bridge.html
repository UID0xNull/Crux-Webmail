🔗 IMAP Bridge — Módulo de Lectura

Adaptador IMAP seguro para lectura de emails a través de Dovecot.

Arquitectura

El IMAP Bridge es un módulo Fastify que expone endpoints RESTful para operaciones de lectura de email. Se conecta a Dovecot vía IMAP sobre TLS.

  [Client] → Fastify API → [IMAP Bridge] → Dovecot IMAP:993
       │                       │                │
       │   /api/emails         │   List/Fetch   │
       │   /api/folders        │   Search       │
       │   /api/email/:id      │   Flags        │
       │                       │                │
       └─── JWT Auth ──────────┴────────────────┘

Endpoints

MétodoEndpointDescripción
GET/api/foldersLista de carpetas del mailbox
GET/api/emails?folder=INBOX&limit=50Paginación de emails
GET/api/email/:idEmail individual con cuerpo completo
GET/api/email/:id/rawCopia cruda (RFC 822)
GET/api/emails/search?q=...Búsqueda IMAP UID SEARCH
PATCH/api/email/:id/flagsModificar flags (seen, flagged, deleted)
PATCH/api/email/:id/moveMover a otra carpeta

Configuración

// Configuración desde .env
DOVECOT_HOST=172.21.0.11
DOVECOT_PORT=993
DOVECOT_SSL_REQUIRED=true
DOVECOT_TIMEOUT=10000
DOVECOT_AUTH_MECHANISMS=plain,login

Seguridad

  • TLS obligatorio: Todas las conexiones IMAP cifradas
  • Auth por usuario: Cada request autenticada con JWT valida per-user mailbox access
  • Sandbox: Solo operaciones de lectura permitidas (READONLY mode)
  • Timeouts: Timeout de 10s en cada operación IMAP

Integración con JMAP

El IMAP Bridge actúa como backend de datos para la capa JMAP. Las peticiones JMAP se traducen a operaciones IMAP equivalentes:

JMAP getMessages() → IMAP UID FETCH
JMAP setMessages() → IMAP STORE FLAGS
JMAP createMessage() → IMAP APPEND
