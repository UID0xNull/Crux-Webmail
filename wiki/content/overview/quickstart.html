⚡ Quick Start Guide

De cero a un sistema de correo funcional en menos de 10 minutos.

Requisitos

  • Docker 24+ y Docker Compose v2+
  • Node.js 20+ (opcional, para desarrollo)
  • Git
  • 4 GB RAM mínimo (8 GB recomendado)
  • Dominio con DNS configurable (para producción)

Paso 1: Clonar y Configurar

git clone https://github.com/user/Crux-Webmail.git
Cd Crux-Webmail
cp .env.example .env
# Editar .env con tu dominio y contraseñas

Paso 2: Crear Secretos

mkdir -p ./secrets
openssl rand -hex 32 > ./secrets/postgres_password.txt
openssl rand -hex 32 > ./secrets/minio_password.txt
openssl rand -hex 32 > ./secrets/jwt_secret.txt
chmod 600 ./secrets/*.txt

Paso 3: Levantar Stack

# Solo en desarrollo (HTTP, sin mTLS)
docker compose -f docker-compose.yml up -d

# En producción (con mTLS, certs, etc.)
docker compose -f docker-compose.yml -f docker-compose.prod.yml up -d

Paso 4: Verificar

# Ver estado de servicios
docker compose ps
# Todos deberían estar (healthy)

# Verificar logs
docker compose logs --tail=50 server
docker compose logs --tail=50 postgres

# Health check
curl -k http://localhost/api/health

Primeros Pasos

  1. Crear usuario admin: curl -X POST http://localhost/api/users -d '{"email":"admin@local","role":"admin"}'
  2. Acceder al dashboard: http://localhost
  3. Verificar monitoring: http://localhost:3000 (Grafana)
  4. Probar envío: telnet localhost 25

Desarrollo Local

# Stack + dev mode hot-reload
docker compose up -d postgres redis postfix dovecot amavis clamav
npm install
npm run dev  # Fastify con watch
