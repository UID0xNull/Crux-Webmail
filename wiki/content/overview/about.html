📖 Acerca del Proyecto

Crux-Webmail: correo seguro, Zero-Trust, moderno.

¿Qué es Crux-Webmail?

Crux-Webmail es un sistema completo de correo electrónico diseñado desde cero con la filosofía Zero-Trust: cada componente, conexión y transacción es verificada y cifrada. Combina:

  • Frontend moderno: Next.js 15 con App Router, SSR/ISR, PWA
  • Backend robusto: Fastify 5 con JMAP, BullMQ, WebSockets
  • Stack de correo legacy: Postfix + Dovecot + Amavis + ClamAV (hardened en Docker)
  • Seguridad: mTLS, TLS 1.3, WebCrypto E2E, sanitización WASM, hardening eBPF
  • Infraestructura: Docker Compose con 16 servicios, 4 redes segmentadas, monitoring completo

Misión

Demostrar que es posible construir un sistema de correo seguro, moderno y auto-hospedado sin comprometer la privacidad, usando las mejores prácticas de 2025.

Stack Tecnológico

Frontend

  • Next.js 15
  • React 19
  • TypeScript 5
  • SWR / Server Actions

Backend

  • Fastify 5
  • BullMQ (jobs)
  • @fastify/secure-session
  • @fastify/websocket

Infra

  • Docker 24+ Compose v2
  • PostgreSQL 17
  • Redis 7
  • Prometheus + Grafana + Loki

Seguridad

  • mTLS / TLS 1.3
  • WebCrypto E2E
  • Sanitización WASM
  • Hardening eBPF

Filosofía de Diseño

  • Zero-Trust: Cada conexión se autentica y cifra, sin confianza implícita
  • Defensa en profundidad: Múltiples capas de seguridad (network → app → data)
  • Minimalismo: Stack mínimo, sin dependencias innecesarias
  • Observabilidad: Monitoring, logging y alerting desde el día 1
  • Reproducibilidad: Docker Compose como infraestructura inmutable
