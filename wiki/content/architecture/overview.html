🏗️ Arquitectura del Sistema — Visión General

Diseño Zero-Trust con microsegmentación, aislamiento de procesos y defensa en capas.

Principios Arquitectónicos

  • Zero Trust: No se confía en ningún servicio, red o usuario — todo se autentica y autoriza
  • Defense in Depth: 5+ capas de seguridad (mTLS, eBPF, WASM sanitización, CSP, rate limiting)
  • Separation of Concerns: Módulos aislados con contratos explícitos (no side-effects compartidos)
  • Immutable Infrastructure: Contenedores read-only, deploy blue-green, rollback automático
  • Observability First: Métricas, logs y traces en cada componente

Flujo de Autenticación

  1. Usuario envía credenciales → Fastify valida contra PostgreSQL (bcrypt)
  2. Backend genera Access Token (JWT, 5min TTL) + Refresh Token (7 días)
  3. Refresh token almacenado en Redis con expiry + hash en DB
  4. Client rota tokens: al expirar access, usa refresh para nuevo access
  5. Todos los endpoints requieren validación JWT (middleware fastify-jwt)
