🔄 Flujos de Datos

Trayectoria de un email: desde recepción hasta visualización en el cliente web.

Flujo de Entrada (Recepción de Email)

  Remot
       SMTP (port 25)
         │
    ▼
┌─────────────┐
│  POSTFIX     │  ← MTA acepta conexión
│  SMTP       │     Greylist check
└──────┬──────┘     Postmaster list validation
       │
    ▼
┌─────────────┐
│  AMAVIS      │  ← Content filter
│             │     ClamAV scan + SpamAssassin
└──────┬──────┘     DKIM verify
       │
    ▼
┌─────────────┐
│  DOVECOT    │  ← LDA entrega a mailbox
│  LDA       │     Maildir format: inbox/.Sent/ etc
└─────────────┘
         │
      /var/mail/{user}/Maildir/

Flujo de Salida (Envío de Email)

  [User en Web] → [Fastify /api/send] → [SMTP Bridge]
                      │                       │
                  Validate JWT              Connect Postfix:25
                  Validate body               MAIL FROM
                  Rate limit                  RCPT TO
                  E2E decrypt (si apply)      DATA + MIME body
                      │                       │
                      │               ┌────────▼────────┐
                      │               │  OpenDKIM milter │ ← Sign Ed25519
                      │               │  Amavis scan    │ ← Antivirus
                      │               │  SPF/DKIM verify│
                      │               └────────┬────────┘
                      │                        │
                      │               ┌────────▼────────┐
                      │               │  Postfix Queue   │ ← Deferred/hold
                      │               └────────┬────────┘
                      │                        │
                      │               ┌────────▼────────┐
                      └───────────────│  Internet SMTP  │ → Destinatario
                                      └─────────────────┘

Flujo de Lectura (Cliente Web → IMAP)

  [Browser] → GET /api/emails?folder=INBOX&limit=50
       │
       ▼
  [Nginx :443] → TLS strip → [Fastify :3000]
                      │
                 JWT Validate
                 Rate Limit
                      │
       ┌──────────────▼──────────────┐
       │         IMAP Bridge         │
       │                             │
       │  1. Connect Dovecot:993    │
       │  2. AUTHENTICATE user      │
       │  3. SELECT INBOX           │
       │  4. UID FETCH 1:* (body)   │
       │  5. Parse headers         │
       │  6. WASM sanitize HTML    │
       │  7. Return JSON          │
       └─────────────────────────────┘
                      │
       ┌──────────────▼──────────────┐
       │        Redis Cache          │ ← LRU TTL 60s
       │  (paginated results)        │
       └─────────────────────────────┘
                      │
       [Browser] ← JSON emails list
       [Next.js]  → Render Email List

Flujo de Autenticación

  1. Login: POST /api/auth/login → body: {email, password}
  2. Verify: Fastify compara bcrypt hash → PostgreSQL
  3. Issue tokens: JWT access (5min) + refresh token (7d) → Redis + DB
  4. Access protected: GET /api/emails + Authorization: Bearer {jwt}
  5. Rotate: Al expirar access, POST /api/auth/refresh → nuevo access + nuevo refresh
  6. Logout: POST /api/auth/logout → revoke refresh tokens en Redis
