🧪 Testing — Jest + ts-jest

Estrategia de testing unit e integración alineada con los scripts de package.json del monorepo.

Scripts Disponibles

El package.json expone tres comandos de testing configurados con Jest:

ScriptComandoDescripción
npm testjest --coverage --detectOpenHandlesTodos los tests + reporte de cobertura + detección de handles abiertos.
npm run test:unitjest --coverage --testPathIgnorePatterns=tests/integrationSolo unitarios, excluye tests/integration.
npm run test:integrationjest --config jest.integration.config.tsIntegración con config dedicada (DB real, Redis real, etc.).

Configuración de Jest

Jest usa ts-jest como transformer para ejecutar código TypeScript directamente sin build previo:

// jest.config.ts (estructura base)
export default {
  preset: 'ts-jest',
  testEnvironment: 'node',
  roots: ['<rootDir>/src'],
  testMatch: ['**/*.test.ts', '**/*.spec.ts'],
  collectCoverageFrom: [
    'src/**/*.ts',
    '!src/**/*.d.ts',
  ],
  coverageThreshold: {
    global: {
      branches: 80,
      functions: 80,
      lines: 80,
      statements: 80,
    },
  },
};

Ejemplo: Unit Test — Auth Service

// src/server/src/services/__tests__/auth.test.ts
import { describe, it, expect, jest } from '@jest/globals';
import { authenticateUser } from '../auth';
import bcrypt from 'bcrypt';

jest.mock('pg');
jest.mock('jsonwebtoken');

describe('authService.authenticateUser', () => {
  it('should return JWT pair on valid credentials', async () => {
    const mockPool = require('pg').Pool;
    mockPool.prototype.query.mockResolvedValue({
      rows: [{
        id: 'usr_001',
        email: 'admin@crux.local',
        password_hash: await bcrypt.hash('secure123', 12),
        role: 'admin',
      }],
    });

    const result = await authenticateUser('admin@crux.local', 'secure123');
    expect(result).toHaveProperty('accessToken');
    expect(result).toHaveProperty('refreshToken');
    expect(result.user.role).toBe('admin');
  });

  it('should throw on invalid password', async () => {
    await expect(
      authenticateUser('admin@crux.local', 'wrong-password')
    ).rejects.toThrow('Invalid credentials');
  });

  it('should lock account after max failed attempts', async () => {
    for (let i = 0; i < 5; i++) {
      await authenticateUser('admin@crux.local', 'wrong').catch(() => {});
    }
    await expect(
      authenticateUser('admin@crux.local', 'correct')
    ).rejects.toThrow('Account locked');
  });
});

Ejemplo: Test de Integración — Fastify Routes

// src/server/tests/integration/auth.routes.test.ts
import { describe, it, expect, beforeAll, afterAll } from '@jest/globals';
import Fastify, { FastifyInstance } from 'fastify';
import authPlugin from '../../src/plugins/auth';

describe('POST /auth/login — Integration', () => {
  let app: FastifyInstance;

  beforeAll(async () => {
    app = Fastify();
    await app.register(authPlugin, {
      database: { uri: process.env.TEST_DB_URL },
      redis: { host: 'localhost', port: 6379 },
    });
  });

  afterAll(async () => { await app.close(); });

  it('should 200 with tokens on valid login', async () => {
    const res = await app.inject({
      method: 'POST',
      url: '/auth/login',
      payload: { email: 'admin@crux.local', password: 'secure123' },
    });
    expect(res.statusCode).toBe(200);
    const body = JSON.parse(res.body);
    expect(body).toHaveProperty('accessToken');
  });

  it('should 429 on rate-limited requests', async () => {
    let res;
    for (let i = 0; i < 11; i++) {
      res = await app.inject({
        method: 'POST',
        url: '/auth/login',
        payload: { email: 'test@test.com', password: 'wrong' },
      });
    }
    expect(res.statusCode).toBe(429); // @fastify/rate-limit
  });
});

Umbrales de Cobertura

┌──────────────────────────────────────────────┐
│          Cobertura Mínima Requerida           │
├────────────┬───────────┬──────────┬──────────┤
│ Líneas     │ Funciones │ Branches │ Stmts    │
├────────────┼───────────┼──────────┼──────────┤
│    80%     │    80%    │   80%    │   80%    │
└────────────┴───────────┴──────────┴──────────┘

🔴 < 80%  → CI falla, bloquea merge
🟡 80-90% → Warning en reporte SonarQube
🟢 > 90%  → Calidad A (quality gate)

Principios de Testing Zero-Trust

  • Unit tests aislados: zero I/O real, todo mockeado.
  • Integration tests validan contratos completos: DB real (PostgreSQL 17 en Docker), Redis 7, Fastify plugins de seguridad activos.
  • --detectOpenHandles en npm test previene memory leaks por conexiones, timers y listeners no cerrados.
  • Seguridad: los tests verifican que middlewares (@fastify/rate-limit, @fastify/jwt, @fastify/helmet) se apliquen.
  • OpenTelemetry: mocks en unit, wiring validation en integration.
