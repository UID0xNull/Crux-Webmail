⚡ Redis 7 (Caché & Sesiones)

Redis como cache layer, session store y backend de BullMQ.

Roles de Redis en el Stack

UsoMóduloImportancia
Sessions@fastify/secure-session🔴 Crítico
Rate Limiting@fastify/rate-limit🔴 Crítico
BullMQ JobsBullMQ queue🔴 Crítico
CacheCache layer general🟡 Alto
Login LockoutBrute-force protection🔴 Crítico

Configuración (docker-compose.yml)

redis:
  image: redis:7-alpine
  container_name: crux-redis
  networks:
    data-network:
  volumes:
    - redis_data:/data
  deploy:
    resources:
      limits:
        cpus: "1"
        memory: 512M
  healthcheck:
    test: ["CMD", "redis-cli", "ping"]
    interval: 10s
    timeout: 3s
    retries: 3

Persistencia

Redis usa RDB snapshots + AOF para durabilidad:

# redis.conf dentro del contenedor
appendonly yes
appendfsync everysec
save 900 1
save 300 10
save 60 10000
