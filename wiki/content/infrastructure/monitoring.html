📊 Monitoring & Observability

Stack Prometheus + Grafana + Loki para métricas, alertas y logs centralizados.

Arquitectura de Observabilidad

  [App Services] ──→ [Prometheus] ──→ [Grafana Dashboards]
       │                   │                    │
       │  metrics/         │  scrape              │
       ▼                   ▼                    ▼
  [Node Exporter]      [Loki] ← log push    [Alertmanager]
       │                       │                   │
       └── /metrics:9100       └── /loki/api/v1   └── Slack/PagerDuty

Prometheus

  • Scrape targets: Todos los contenedores exponen /metrics
  • Intervalo: 15s para app, 30s para infraestructura
  • Retención: 30 días en volumen persistente

Loki — Logs Centralizados

  • Log collector: Promtail en cada contenedor
  • Retención: 30 días hot, 90 días cold (MinIO)
  • Query language: LogQL con stream selector y label filters

Alertmanager

routes:
  - match:
      severity: critical
    receiver: slack-critical
  - match:
      severity: warning
    receiver: slack-warnings

receivers:
  - name: slack-critical
    slack_configs:
      - channel: '#alerts-critical'
  - name: slack-warnings
    slack_configs:
      - channel: '#alerts-warning'
