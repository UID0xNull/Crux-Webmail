🛡️ Política de Seguridad Zero-Trust

Arquitectura de confianza cero: nada se confía, todo se verifica.

Principios Fundamentales

Zero-Trust significa que ningún actor interno ni externo es confiable por defecto. Cada solicitud, usuario, servicio y red es autenticado, autorizado y cifrado de forma explícita.
  • Verificación explícita: Toda solicitud requiere autenticación JWT + binding de sesión + fingerprint del dispositivo.
  • Acceso mínimo privilegiado: Solo las capabilities necesarias por contenedor (cap_drop ALL + cap_add explícitos).
  • Microsegmentación de red: 4 redes Docker aisladas — public, app, data, monitor — sin tránsito cross-network.
  • Cifrado en tránsito y en reposo: TLS 1.2+ obligatorio, mTLS entre servicios internos, secrets de Docker para credenciales.
  • Supervisión continua: Prometheus + Loki + Grafana para telemetría, audit logs JSON estructurados, alertas automáticas.
  • Hardening profundo: Filesystems read-only, seccomp profiles, no-new-privileges, tmpfs para datos volátiles.

Capas de Seguridad

Capa Tecnología Propósito
Perímetro Nginx Edge proxy: TLS termination, rate-limiting, CSP headers, HTTP→HTTPS redirect
Aplicación Fastify middleware JWT auth, CSRF, CSP nonce-based, security headers, Prometheus metrics
Red Docker networks 4 subredes aisladas, solo tráfico autorizado entre capas
Contenedor Seccomp + Capabilities Syscalls permitidas explícitamente, cap_drop ALL, read_only filesystem
Datos Docker Secrets + Postgres Credenciales nunca en texto plano, data checksums, pg_hba.conf estricto
Correo DKIM + DMARC + SPF Autenticación de remitente, firma criptográfica, policy reject
Contenido Amavis + ClamAV Filtrado anti-spam, escaneo antivirus de attachments

Matriz de Riesgos

Riesgo Mitigación Severidad
Brute-force en login Rate limit 5 req/s en auth, auto-ban tras 3× threshold Alta
CSRF Double-submit cookie, token por sesión, validación en POST/PUT/PATCH/DELETE Alta
XSS CSP nonce-based estricto, frame-ancestors none, sanitize inputs Alta
Fuga de secretos Docker secrets, nunca .env hardcodeado, rotation automática Crítica
Contenedor comprometido Seccomp profile, read_only, no-new-privileges, cap_drop ALL Media
Phishing /Spoofing DKIM ed25519, DMARC reject, SPF -all Media
DDoS Rate-limiting en nginx y Redis, connection limit per IP Alta
