📨 Autenticación de Correo: DKIM, DMARC y SPF

Firmas criptográficas y políticas DNS para prevenir spoofing y phishing.

DKIM — DomainKeys Identified Mail

Firma criptográfica del contenido del email usando llaves asimétricas. El receptor verifica la firma con la clave pública publicada en DNS.

DKIM_SELECTOR=default
DKIM_ALGORITHM=ed25519

# DNS Record:
# default._domainkey.crux.local. TXT v=DKIM1; k=ed25519; p=...
  • Algoritmo Ed25519 (moderno, resistente a ataques) en lugar de RSA heredado
  • Selector default en ._domainkey.crux.local
  • Firma se aplica en Postfix vía milter de OpenDKIM

SPF — Sender Policy Framework

Lista blanca de servidores autorizados para enviar correo en nombre del dominio.

# DNS Record:
crux.local. TXT v=spf1 mx a -all

SPF_POLICY=-all  # "Hard fail": rechazar si no coincide
  • mx: Autorizar el servidor MX del dominio
  • a: Autorizar la dirección A del dominio
  • -all: Rechazar cualquier otro origen (soft fail ~all solo para staging)

DMARC — Domain-based Message Authentication

Política de comportamiento cuando SPF y/o DKIM fallan. Instrucciones claras para el receptor.

# DNS Record:
_dmarc.crux.local. TXT v=DMARC1; p=reject; rua=mailto:dmarc@crux.local; rf=afrf

DMARC_POLICY=reject
Valor Efecto Uso
none Sin acción, solo reporte Desarrollo / Debug
quarantine Enviar a spam/quarantine Transición
reject Rechazar el mensaje Producción (hardened)

DNSSEC — Seguridad de DNS

DNSSEC_ENABLED=true
  • Firma criptográfica de los registros DNS del dominio
  • Previene envenenamiento de cache DNS y DNS hijacking
  • Verificación chain of trust desde root TLD hasta el registro DKIM/SPF/DMARC
