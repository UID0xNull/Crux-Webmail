🔑 Gestión de Secretos

Docker Secrets + vault integration para credenciales en producción.

Docker Secrets

Todas las credenciales sensibles se gestionan como Docker Secrets, montados en /run/secrets/ dentro de cada contenedor. Nunca aparecen en texto plano en docker-compose.yml ni en .env.

# docker-compose.yml
secrets:
  postgres_password:
    file: ./secrets/postgres_password.txt
  minio_password:
    file: ./secrets/minio_password.txt
  grafana_password:
    file: ./secrets/grafana_password.txt
  mail_server_tls_key:
    file: ./secrets/tls_key.pem
  mail_server_tls_cert:
    file: ./secrets/tls_cert.pem

Consumo de Secretos

Cada servicio consume secretos vía variable de entorno con sufijo _FILE:

# PostgreSQL
environment:
  POSTGRES_PASSWORD_FILE=/run/secrets/postgres_password
secrets:
  - postgres_password

# MinIO
environment:
  MINIO_ROOT_PASSWORD_FILE=/run/secrets/minio_password
secrets:
  - minio_password

# Grafana
environment:
  GF_SECURITY_ADMIN_PASSWORD_FILE=/run/secrets/grafana_password
secrets:
  - grafana_password

Jerarquía de Secretos

Nivel Tipo Ejemplo Ciclo de vida
Infraestructura TLS/SSL Certs tls_key.pem, tls_cert.pem Rotación diaria (7d validity)
Base de datos DB Credentials postgres_password.txt Rotación mensual
Almacenamiento S3/MinIO Access minio_password.txt Rotación mensual
Monitoring Dashboard Access grafana_password.txt Rotación trimestral
Aplicación JWT Secrets JWT_SECRET, JWT_REFRESH_SECRET Rotación semestral
Sesión CSRF tokens Redis keys csrf:* 1 hour TTL

Reglas de Seguridad

  • 🔒 .gitignore: El directorio secrets/ está siempre en .gitignore
  • 🔐 Mínimo de 64 chars: Los JWT secrets requieren mínimo 64 caracteres (Zod validation)
  • 🔄 Rotación automática: Cron job para rotar certificados diarios, secretos mensuales
  • 📝 Audit logging: Todo acceso a secretos se registra en el audit log de Loki
  • 🚫 No hardcode: Si se detecta un secreto hardcodeado en el código, el linter falla en CI
