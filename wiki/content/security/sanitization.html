🧼 Sanitización de Contenido con WASM

Procesamiento seguro de contenido HTML en emails aislado mediante WebAssembly.

Problema

Los emails HTML contienen scripts, iframes, event handlers y estilos potencialmente peligrosos que pueden ejecutarse en el navegador del cliente. La sanitización es fundamental para prevenir:

  • XSS (Cross-Site Scripting): Scripts embebidos en emails
  • Phishing: Redirecciones maliciosas, formularios falsos
  • Resource Abuse: Imágenes trackeadas, requests externos
  • CSS Injection: Estilos que manipulan layout del cliente

Solución: WASM Sandbox

El WASM (WebAssembly) permite ejecutar código de sanitización en un sandbox aislado, con acceso cero al DOM ni a APIs del navegador. El contenido se procesa en un entorno controlado antes de renderizar.
Capacidad Implementación Beneficio
Aislamiento WASM sandbox nativo Sin acceso al DOM, localStorage, cookies, ni fetch
AST Parser htmlparser2 en WASM Parsing estructural del HTML sin interpretación
Allowlist de tags Filtro tag+attribute Solo tags seguros: p, span, img (self), a (http)
Image handling Blob/URI transform Imágenes externas reemplazadas por placeholder
CSS sanitization Style parser estricto Solo propiedades CSS seguras, sin expression() ni @import
