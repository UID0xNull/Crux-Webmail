🔐 Cifrado End-to-End con WebCrypto

Criptografía del lado del cliente usando la Web Crypto API nativa del navegador.

Arquitectura de Cifrado

El cifrado E2E garantiza que el contenido de los emails nunca se descifra en el servidor. Las claves residían exclusivamente en el cliente (navegador), protegidas por el keystorage de WebCrypto con extractable: false.

Stack Criptográfico

Algoritmo Uso Biblioteca
AES-GCM 256 Cifrado de contenido (simétrico) Web Crypto API nativo
ECDH P-256 Key exchange (asimétrico) Web Crypto API nativo
Salted Argon2id Key derivation de passphrase openpgp-js / Crypto-JS
SHA-256 / SHA-512 Fingerprints, verificaciones Web Crypto API (SubtleCrypto)
Ed25519 Firmas DKIM y verificación DHparams en DNS

Dependencias Criptográficas

  • Crypto-JS (v4.2.0): Utilidades criptográficas generales (hashing, HMAC)
  • OpenPGP.js (v6.1.0): Implementación PGP para firma/verificación de emails
  • Web Crypto API: AES-GCM, ECDH, SHA-256 — nativo del navegador
